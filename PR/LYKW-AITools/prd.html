1. 背景与目标

1.1 背景

用户在麟云开物完成支付后，具备为商家裂变传播、沉淀口碑的动机；商家侧需要低门槛产出宣传素材并可控地发放激励（红包、优惠券等）。通过 AI 降低内容生产成本，通过支付场景自然触达用户，形成「支付成功 → 推广任务 → 商家确认 → 激励发放」闭环。

1.2 产品目标

  • 用户侧：支付成功页提供明确入口，一键生成可保存、可转发的推广视频与图文素材；参与点赞打卡等轻量任务；完成后有机会获得红包/优惠券等权益。
  • 商家侧：在「我的」中进入 AI 助手，维护店铺素材库，按参数生成宣传视频与文案，审核后可用于活动或对用户侧任务进行激励配置。
  • 平台侧：增强支付后黏性、提升商家活跃度与营销转化数据可追踪性（具体指标见第 8 节）。

1.3 范围说明

  • 本期重点：用户端支付成功后的推广链路、商家端 AI 助手入口及素材与生成能力、商家对用户推广行为的确认与激励发放。
  • 暂不展开（可单列二期）：与第三方短视频/社交平台深度 OAuth、复杂投放托管、数字人直播等。

2. 名词与角色

术语 定义
用户（C 端） 在商家场景下完成支付的消费用户
商家（B 端） 使用麟云开物收款的入驻商户
推广任务 发视频、点赞打卡等由平台/商家侧配置的用户可参与行为
素材库 商家在 AI 助手中维护的店铺介绍、图片、视频等，用于 AI 生成
商家确认 商家对用户完成的推广行为或待发放激励进行审核/确认的动作

3. 用户端需求（手机端）

3.1 入口与触达

需求 ID 描述 优先级
U-001 支付成功页面底部展示宣传框，文案示例：「帮商家推广，领红包」（具体文案可配置或由商家在活动中设定） P0
U-002 点击宣传框进入推广入口页，展示当前可用的推广方式入口（如：发视频、点赞打卡等），列表可根据商家/活动配置展示或隐藏 P0
U-003 未登录/会话过期时，引导登录或与订单身份绑定，避免冒领激励（策略与支付账号一致原则） P0

3.2 推广入口页（聚合页）

需求 ID 描述 优先级
U-010 页面展示任务说明、奖励规则摘要（红包/券的面额或概率区间说明需合规展示） P0
U-011 卡片/按钮：发视频、点赞打卡（及后续可扩展任务位） P0
U-012 展示任务完成状态、审核中、已发奖等状态（与商家端确认流联动） P0

3.3 发视频（一键 AI 生成推广视频）

需求 ID 描述 优先级
U-020 进入后自动或一键触发基于当前订单/商家关联素材生成推广短视频（素材来源见商家端「素材库」） P0
U-021 生成过程展示进度或排队状态；失败时提示原因并支持重试 P0
U-022 生成结果支持预览、保存到相册（或系统分享） P0
U-023 提供一键转发能力：调用系统分享面板到微信等（具体渠道受系统与应用权限限制） P1
U-024 用户标记「已转发」或上传凭证策略（若业务要求防刷）：可与分享回调、截图上传、或诚信声明三选一或组合，需在合规与体验间平衡 P1
U-025 同一订单/同一任务的重复生成次数限制（防滥用，与商家成本策略相关） P1

3.4 点赞打卡（一键生成转发文案与图片）

需求 ID 描述 优先级
U-030 进入后一键生成适合社交平台转发的文案 + 配图（图文卡或海报） P0
U-031 文案支持多风格简短版切换（可选，若一期只做一种风格可列为 P1） P1
U-032 生成结果支持预览、保存图片、复制文案 P0
U-033 一键转发（同 U-023，系统分享） P1
U-034 打卡/完成状态上报，进入「待商家确认」 P0

需求 ID 描述 优先级
U-040 用户完成指定动作后，状态进入待商家确认 P0
U-041 商家确认通过后，按规则发放红包/优惠券等（与支付平台营销能力对接） P0
U-042 用户侧展示到账/券到账通知及使用说明 P0
U-043 商家驳回或超时未处理时的用户提示与申诉/客服入口（策略待定） P1

4. 商家端需求（手机端）

4.1 入口

需求 ID 描述 优先级
M-001 路径：我的 → 应用菜单 → AI 助手，进入 AI 助手首页 P0
M-002 权限：仅商家主账号或具备「营销/AI助手」子权限的账号可见（具体 RBAC 与现有商家后台对齐） P0

4.2 AI 助手首页结构（信息架构）

建议分块（tabs 或纵向模块均可，UI 实现待定）：

  1. 店铺素材与资料（为后续一键生成提供输入）
  2. AI 宣传视频生成（商家主动生成，带参数）
  3. 用户推广待办 / 确认与发奖（与 C 端任务闭环）
  4. 可选：消耗记录、额度说明（若 AI 按量计费）

4.3 店铺素材与资料

需求 ID 描述 优先级
M-010 编辑/上传店铺介绍（文本）：店名、卖点、地址摘要、主营品类等 P0
M-011 上传图片：门面、菜品/商品、环境、Logo 等，支持多张，设封面/主图标记 P0
M-012 上传视频：店内实拍、老板证言等短视频，作为 AI 参考或混剪素材 P1
M-013 缺失项提示：若关键字段为空，在用户端生成前提示商家补全（可软拦截） P1
M-014 素材合规提示：用户上传内容的版权与肖像权责任条款确认（勾选/协议） P0

4.4 AI 宣传视频生成（商家侧）

需求 ID 描述 优先级
M-020 基于素材库一键或分步生成宣传短视频 P0
M-021 可配置参数（一期建议最小集）：
• 视频风格（如：温馨种草 / 促销快闪 / 极简高级感 等枚举）
• 时长（如：15s / 30s / 60s 或区间）
• 画幅（9:16 竖版优先，兼容 1:1）
• 字幕开关、口播/配乐风格（若技术可选项） 		P0
M-022 生成队列与失败重试；额度或次数展示 P0
M-023 生成结果预览；商家确认采纳 / 重新生成 / 下载 P0
M-024 已采纳视频可加入「用户推广可用素材池」或标记为默认模板（供 U-020 消费） P1

4.5 AI 转发文案与配图（商家侧，可选与 C 端对齐）

需求 ID 描述 优先级
M-030 商家可预生成多套「打卡用」文案+海报，供 C 端一键调用，或仅由 C 端实时生成（二选一或并存，待评审） P1

4.6 用户推广确认与激励发放

需求 ID 描述 优先级
M-040 列表展示待确认的用户推广记录：关联订单、任务类型、提交时间、凭证（若有） P0
M-041 一键确认通过；支持单条驳回并填写原因（可选） P0
M-042 确认通过后，一键发红包 / 发优惠券（调用现有营销中心或红包接口，金额/券批次需前置配置） P0
M-043 批量操作、筛选（按日期、任务类型）（P1） P1
M-044 防刷策略配置入口或与平台风控策略对齐（频率、同设备、同收款单等） P1

5. 核心业务流程（简述）

5.1 用户发视频 / 打卡

流程图源码（Mermaid）：可在支持 Mermaid 的编辑器中渲染。

flowchart LR
  A[支付成功页] --> B[推广入口]
  B --> C{任务类型}
  C -->|发视频| D[AI 生成视频]
  C -->|点赞打卡| E[AI 文案+配图]
  D --> F[保存/转发]
  E --> F
  F --> G[提交完成]
  G --> H[待商家确认]
  H --> I[商家确认]
  I --> J[发红包/券]

5.2 商家维护与生成

商家在 AI 助手中维护素材 → 配置视频参数并生成 → 商家确认成片 →（可选）进入用户侧可用素材池；并行处理用户推广待办列表并完成发奖。

6. 非功能需求

类别 要求
性能 视频生成异步化；列表页分页；关键操作有明确等待反馈
可用性 手机端单手操作友好；步骤不超过用户心理预期（一键优先）
安全 商家与用户数据隔离；生成内容审计日志；激励发放与支付风控联动
合规 营销活动规则公示；不得以虚假进度诱导分享；红包/券需符合金融监管与平台规则
兼容 iOS / Android 主流版本；分享、存相册权限按需申请

7. 依赖与接口（待技术评估）

  • 支付成功页组件化配置能力（宣传框开关、文案、跳转参数）。
  • AI 视频/图文生成服务（自建或三方）、存储与 CDN。
  • 红包与优惠券发放能力、营销预算与商户账户。
  • 消息通知（用户侧到账提醒、商家侧新待办提醒）。

8. 成功指标（建议）

指标 说明
支付成功页推广入口点击率 衡量触达效果
任务启动率、完成率 发视频 / 打卡分别统计
商家确认时效、发奖成功率 运营与体验
商家 AI 助手周活跃、素材完整度 B 端采纳
投诉与风控拦截率 合规与健康度

9. MVP 与二期划分（建议）

阶段 内容
MVP 支付成功宣传框 + 推广入口；发视频与点赞打卡两任务；C 端生成、保存、系统分享；商家素材库（文图）；商家侧视频生成+参数（风格、时长、画幅）；待办确认+一键发红包/券
二期 更多任务类型；商家预置模板库；分享归因与深度平台对接；批量审核；更细风控与 A/B 文案

10. 待决策项（Open Questions）

  1. 用户「转发成功」的判定标准：仅用户自述、系统分享回调、还是必须回传链接（各端能力不同）。
  2. 红包/优惠券成本由平台补贴、商户承担还是按比例分摊。
  3. AI 生成次数/时长套餐：是否纳入商户套餐或单独计费。
  4. 商家采纳的视频是否强制作为 C 端唯一素材源，还是 C 端始终实时拼装素材库。
