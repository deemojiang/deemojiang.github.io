# 交互原型工作流（本仓库约定）

> 在 **`调研文档/原型工作流.md`** 中有一份相同正文的副本，便于与其它调研材料放一起查阅。

本文记录在本项目中画原型、适配与发布时的固定做法，**以后同类项目可复用同一套方式**，减少来回沟通与返工。

---

## 1. 左侧原型列表（站点地图）

- **位置**：`index.html` 左侧 `.sitemap`，按「用户端 → 管理端手机 → 管理端电脑 → 宣传物料」等分区，每一项对应一个可跳转的 **屏幕块**（`.screen`）。
- **跳转方式**：链接使用 `href="#屏幕id"`，由 `prototype.js` 里 `navigateTo` / `showScreen` 切换 **仅激活对应 `#id` 的 `.screen`**，并更新地址栏 hash。
- **约定**：
  - 每个原型画面一个稳定 **`id`**（如 `m-list-zjj`、`pc-list`），命名尽量见名知义。
  - 需要「带状态进入」的详情（如 PC 工单详情待办/已办）用 **独立函数**（如 `goPcDetail('done')`）在进入前设好 class / 文案，再 `showScreen`，避免侧栏点进详情时状态错乱。

---

## 2. 适配手机端 / 电脑端 / 广告牌

- **多预览区**：`index.html` 内用不同容器包同一套页面结构或子集，例如：
  - `#preview-phone-wrap`：手机框 + 竖屏内容；
  - `#preview-pc-wrap`：浏览器 chrome + 宽屏；
  - `#preview-billboard-wrap`：5:10 竖版海报（无手机壳）。
- **按屏切换设备**：侧栏链接上 **`data-device="phone" | "pc" | "billboard"`**，在 `prototype.js` 的 `setDevice` 里控制显示哪一个预览容器，避免三端同时抢版面。
- **样式拆分**：
  - 全局样式集中在 **`prototype.css`**；
  - 手机与 PC 差异尽量用 **class 修饰**（如 `.pc-screen`、列表宽表）或 **媒体查询**，少复制整段 HTML。
- **手机端交互原则**：
  - 列表可保持 **紧凑行**（一行 meta + 一行标题 + 状态角标），详情再展开字段；
  - 与 PC 能力不一致时（如「仅 PC 提供筛选」）在原型里 **直接去掉手机筛选区**，避免误解需求。

---

## 3. 行为与校验（`prototype.js`）

- **导航**：统一走 `navigateTo`；对需要默认状态的特殊屏（如 `pc-detail`、`m-detail-zjj`）在 `navigateTo` 里 **分支调用初始化函数**，保证从侧栏进入也是正确初始状态。
- **表单**：轻量校验可放在同一脚本内；复杂校验保持可读函数，避免和业务混成一团。

---

## 4. 同步到博客（静态托管）

- **本地博客仓库路径**（本机约定）：与工程并列的 `deemojiang-blog`（GitHub Pages：`deemojiang.github.io`）。
- **同步规则**：将本项目 **`FeedbackPlatform` 文件夹** 全量覆盖到博客内 **`PR/FeedbackPlatform/`**（与其它项目一样放在 `PR/` 下，**文件夹名与仓库内项目同名**）。
- **发布步骤**（在 `deemojiang-blog` 目录）：
  1. `git status` 确认变更；
  2. `git add PR/FeedbackPlatform/…`；
  3. `git commit -m "说明本次原型变更要点"`；
  4. `git push origin main`。
- **网络失败时**：提交已在本地完成，待网络恢复后**补执行** `git push` 即可。

---

## 5. 以后「同类问题」怎么处理（检查清单）

| 步骤 | 做什么 |
|------|--------|
| 1 | 新画面 → 在左侧 sitemap 增加入口 + 对应 `.screen#id` |
| 2 | 需要多状态 → 用函数封装进入逻辑，侧栏 `navigateTo` 里必要时重置状态 |
| 3 | 手机 / PC 差异 → 删繁就简展示 + CSS 分端修饰，避免两端强行同一套复杂表格 |
| 4 | 视觉问题 → 先查 **父级 flex/grid、百分比宽度、aspect-ratio**，避免「外框被拉成长条」这类布局坑 |
| 5 | 定稿一段落后 → **复制到 `deemojiang-blog/PR/FeedbackPlatform`** 并 **commit + push** |

---

## 6. 相关文件

| 文件 | 作用 |
|------|------|
| `index.html` | 左栏导航 + 各端预览与 `.screen` 结构 |
| `prototype.css` | 全局与分端样式 |
| `prototype.js` | 导航、设备切换、列表筛选等交互 |
| `billboard.html` | 独立打开的广告牌单页（可选与预览区同源结构） |

若将本工作流迁移到其他原型项目：保留 **「左栏 + 多屏 + 多设备容器 + 单一样式表/脚本 + PR 目录同步博客」** 的结构即可快速对齐习惯。
