2. 项目概述

2.1 背景与目标

建设一套面向居民的物业问题上报与政务协同办理平台：居民通过手机扫码即可提交问题；由住建局统一分流、分派或自办；乡镇/街道管理员按权限处理问题并闭环；管理侧支持移动端与电脑端协同，电脑端提供数据统计与监管视图。

2.2 产品形态

端设备形态核心用户核心目标
用户端手机（浏览器/H5/小程序等，实现方式待定）居民/报修人扫码进入、填写并提交问题反馈
管理员手机端手机住建局管理员、乡镇/街道管理员登录后分派或办理问题（与电脑端功能对齐，交互适配小屏）
管理员电脑端桌面浏览器住建局管理员、乡镇/街道管理员与手机端管理功能一致，并补充 数据统计与报表

3. 角色与权限

3.1 角色定义

角色说明主要能力
普通用户（居民）无需登录扫码进入反馈页、提交问题（含图片等）
住建局管理员需登录分流：可将工单分派至四街道（龙山/雉城/画溪/太湖，全称、单选），或自行办理；查看所辖范围数据与统计
乡镇/街道管理员需登录办理已分派至本街道的工单；可见范围与账号「管辖街道」配置一致

3.2 账号规模（一期）

  • 管理端账号共 5 个（可配置：如 1 个住建局账号 + 4 个乡镇/街道账号，具体归属以业务确认表为准）。
  • 登录凭证形式：账号 + 密码；须支持通过短信验证码的忘记密码/重设密码（见 Markdown 5.7）。

3.3 权限矩阵（摘要）

功能普通用户住建局管理员乡镇/街道管理员
提交问题反馈✅——
查看全部工单—✅（按产品规则限定范围）仅本乡镇相关工单
分流/分派至街道—✅—
自行办理（住建局）—✅—
办理/办结（乡镇）——✅
管理端数据统计（电脑端）—✅（建议）按权限可见本乡镇或全局，以实现时配置为准
账号管理（电脑端）—✅（系统管理员/授权角色）—

3.4 分派后通知（摘要）

分派至某街道后，向该街道管理员登记手机号发送短信提醒处置（见 Markdown 3.5）。

4. 核心业务流程

4.1 居民上报流程

  1. 用户扫描入口二维码，进入 问题反馈界面（用户端）。
  2. 填写表单并提交。
  3. 系统生成一条 问题反馈工单，节点进入 待分派（见第 6 节）。
  4. 用户在提交成功页可仅做 成功提示；本期不要求用户端查询历史工单。

4.2 住建局分流流程

  1. 住建局管理员登录管理端（手机或电脑）。
  2. 查看待分派/待办列表中的工单。
  3. 分流决策：分派至四街道之一（管理侧无「其他」；居民选「其他」的工单须人工/规则映射），确认后发短信通知该街道管理员；或 自行办理。
  4. 工单状态随操作流转（见第 6 节）。

4.3 乡镇/街道办理流程

  1. 乡镇管理员登录管理端。
  2. 在待办列表中查看 已分派至本乡镇 的工单。
  3. 办理：必填 办理意见；选填 图片（多张）。
  4. 办理结束：提交后工单标记为 已办结。

5. 功能需求详述（节选）

5.1 用户端 — 问题反馈表单

字段必填说明
问题描述是文本
所在街道是下拉：龙山、雉城、画溪、太湖、其他（选「其他」无需另填）
小区名称是文本
姓名 / 电话否文本
图片否多张

5.5 管理员电脑端 — 数据统计

概览（总量/已办/待办）、按乡镇/街道分布、按办理时长（24 小时、3 天、7 天、超 7 天）等，口径以开发联调为准。

5.6 账号管理、5.7 安全（摘要）

电脑端 账号管理：维护角色与管辖街道（四街道单选、无「其他」）、重置密码经短信。安全：8 位以上且含大小写+数字+特殊符号；忘记密码经短信验证码；输错 3 次锁 5 分钟。全文见 Markdown 5.6 / 5.7。

6. 工单节点（建议）

详情时间轴建议四级节点（顺序固定）：

节点说明
用户提交居民已完成反馈提交
待分派住建局尚未完成分派（四街道或自行办理路径）
处理中责任主体正在办理
办结结案
